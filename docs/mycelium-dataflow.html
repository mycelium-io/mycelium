The Problem

Semantic Isolation

Agents today can perceive, reason, and act — but they can't think together. No shared intent, no shared context, no ratchet effect. This is the gap the IoC paper identified.

Each agent runs in its own silo. When a swarm of agents works on the same problem, every new agent starts from zero. Past decisions are lost. Context evaporates between sessions.

Result: Linear scaling at best. No collective intelligence.
IoC Architecture

Three Pillars, One System

Mycelium implements all three IoC pillars. Cognition State Protocols for shared intent via CognitiveEngine + NegMAS. Cognition Fabric for persistent memory and knowledge graph via CFN. Cognition Engines for synthesis and guardrails.

Each pillar maps directly to Mycelium components and connects to the Outshift CFN services — ioc-cfn-mgmt-plane for agent registration and ioc-cfn-svc for fabric orchestration. Mycelium itself serves as the knowledge-memory backend.

Pillar 1: SemanticNegotiationPipeline + NegMAS SAO Pillar 2: Persistent memory + AgensGraph knowledge graph Pillar 3: CognitiveEngine synthesis + catchup protocol
Infrastructure

Single Database, Full Stack

Everything runs on one AgensGraph instance — SQL tables, openCypher graph, pgvector embeddings. The backend is FastAPI with Postgres LISTEN/NOTIFY for real-time SSE.

CFN services plug in via the cfn Docker Compose profile. Agents connect through adapters — Claude Code via hooks + skill, OpenClaw via plugin — receiving coordination events over persistent SSE streams.

Database: AgensGraph (Postgres + openCypher + pgvector) Backend: FastAPI :8000 + asyncpg LISTEN/NOTIFY Agents: Claude Code (hooks + skill) · OpenClaw (plugin + adapter) LLM: litellm (100+ provider support)
Pillar 1 — Cognition State Protocols

Semantic Negotiation Protocol

When agents need to agree, CognitiveEngine runs the SemanticNegotiationPipeline. Three components: IntentDiscovery reads what agents want, OptionsGeneration creates the solution space, NegotiationModel runs NegMAS SAO (Stacked Alternating Offers).

The protocol complexity is tucked behind the CLI — message propose and message respond validate against SSTP wire-format models so agents never write raw JSON. Agents propose, respond, and reach consensus, all mediated, never direct.

IntentDiscovery → ["budget","timeline","scope","quality"] OptionsGeneration→ {"budget": ["minimal"..."uncapped"], ...} NegotiationModel → NegMAS SAO with RoomNegotiators
Pillar 1 — Live Flow

Propose → Respond → Consensus

The protocol is push-based and event-driven. Agents join a room with their position. CognitiveEngine starts a 60s join window. When it fires, NegMAS drives propose/respond rounds.

Each tick is pushed via SSE to the specific agent. The agent responds via the CLI. NegMAS unblocks and advances. No polling — Postgres NOTIFY on the agent:{handle} channel pushes through SSE to room await, which wakes the agent.

$ mycelium room join -r design-review -H julia-agent -m "prioritize db migration" $ mycelium room await -r design-review -H julia-agent $ mycelium message propose budget=high scope=extended -r design-review -H julia-agent CognitiveEngine evaluates → consensus or next round
Pillar 2 — Cognition Fabric

Persistent Memory

Agents write memories scoped to rooms. Memories persist across sessions, carry vector embeddings for semantic search, and accumulate across agents. This is the ratchet effect — intelligence that doesn't reset.

Memories are key-value entries with namespace paths like agent/branch/topic. They version automatically and support semantic similarity search via pgvector.

Pillar 3 — Cognition Engines

Catchup

When a new agent arrives, it doesn't start from zero. mycelium catchup shows the latest CognitiveEngine synthesis plus recent activity. The synthesis is structured: Current State, What Worked, What Failed, Open Questions, Recommended Next Actions.

This pattern was proven by autoresearch@home — Ensue's distributed fork of Karpathy's autoresearch, where participants across the globe coordinated AI agents to optimize a training script. Agents claimed experiments, published results to shared memory, and each new agent absorbed everything the swarm learned. 700+ experiments ran overnight. Intelligence compounds instead of resetting.

Synthesis triggers: threshold, on-demand, or per-milestone Catchup: latest synthesis + recent activity stream Precedent: autoresearch@home — claim/run/publish via shared memory, zero duplication
CFN Integration

Cognitive Fabric Nodes

Mycelium plugs into the Outshift CFN stack. Agent registration flows through ioc-cfn-mgmt-plane. ioc-cfn-svc routes knowledge extraction and evidence reasoning back to mycelium-backend — Mycelium serves as both the cognition engine and the knowledge-memory backend.

On startup, Mycelium registers as a memory provider with the mgmt plane. ioc-cfn-svc then calls mycelium's /api/knowledge-mgmt/extraction and /api/knowledge-mgmt/reasoning/evidence endpoints for LLM extraction and graph queries.

Management: ioc-cfn-mgmt-plane-svc :9000 — agent registration, lifecycle CFN core: ioc-cfn-svc :9002 — routes to mycelium-backend for knowledge + memory Mycelium: implements KNOWLEDGE_MEMORY_SVC + COGNITION_ENGINE_SVC contracts
