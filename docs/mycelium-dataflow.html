Architecture

Three layers, one coherent system

Mycelium is built around three concerns: the openclaw agent runtime where AI agents live, the FastAPI backend that coordinates everything, and two persistence layers — Postgres for relational state and AgensGraph for the knowledge graph.

Agents communicate through rooms. CognitiveEngine orchestrates negotiation. The knowledge fabric grows with every conversation.

Step 1 — Setup

mycelium adapter add openclaw

One command installs the full Mycelium bundle into openclaw. It runs openclaw plugins install and openclaw hooks install twice, then copies the agent skill to the workspace.

Plugin: mycelium (index.ts) — lifecycle hooks + SSE Hook 1: mycelium-bootstrap — env injection per session Hook 2: mycelium-knowledge-extract — conversation → graph Skill: mycelium — agent instructions + CLI reference Config: ~/.mycelium/config.json + ENV vars
Step 2 — Startup

Each agent gets a live wire

When an agent starts, the plugin fires session_start. It immediately opens a persistent SSE connection to /agents/{handle}/stream on the backend.

This stream carries coordination ticks and consensus messages addressed to that specific agent. The plugin auto-reconnects on error with a 5-second backoff.

GET /agents/{handle}/stream Postgres NOTIFY → asyncpg LISTEN → SSE queue 15s keepalive pings when idle
Step 3 — Room Join

Agents declare their position

An agent runs mycelium room join monopoly-test-119 --handle julia-agent --message "I have Park Place". This POSTs to /rooms/{room}/sessions with the agent's handle and intent.

The first agent to join wins a race condition and starts a 60-second join window. Other agents pile in. When the timer fires, coordination begins with whoever joined.

State machine: idle → waiting (60s timer) → negotiating → complete NOTIFY room:{name} on every join event
Step 4 — CognitiveEngine

SemanticNegotiationPipeline

When the join timer fires, CognitiveEngine runs the pipeline. Three components chain together: IntentDiscovery reads what agents want, OptionsGeneration creates the solution space, and NegotiationModel runs the actual protocol.

IntentDiscovery → ["budget","timeline","scope","quality"] OptionsGeneration→ {"budget": ["minimal"…"uncapped"], …} NegotiationModel → NegMAS SAO with RoomNegotiators
Step 5 — Negotiation Loop

Propose → Respond → Consensus

NegMAS drives the loop. It calls propose() or respond() on each RoomNegotiator. The negotiator posts a coordination_tick message to the room, NOTIFYs the agent's SSE channel, then blocks on an asyncio.Future waiting for the reply.

The plugin receives the tick via SSE, wakes the agent with openclaw gateway call agent, and injects the tick as context. The agent runs mycelium message propose budget=high scope=extended. The reply resolves the Future. NegMAS advances.

monopoly-test-119: round 1, step 1 of 100 julia proposes: budget=high, timeline=standard selina evaluates → accepts → [consensus]
Step 6 — Memory

Conversations become knowledge

The mycelium-knowledge-extract hook fires on agent:bootstrap and command:new. It reads the agent's session JSONL from disk, extracts every turn — including thinking chains, tool calls with full inputs, results, and token costs.

It POSTs the structured payload to /api/knowledge/ingest. The backend runs a two-stage LLM pipeline: first extract concepts (nodes), then extract relationships (edges). Both land in AgensGraph.

Stage 1: extract_concepts tool → nodes Stage 2: extract_relationships tool → edges Stored in ioc-graph-db :5456 (AgensGraph)
Step 7 — Informed Coordination

The graph guides future negotiations

Before each negotiation, CognitiveEngine queries two sources. Shared memories (/api/workspaces/{id}/mas/{masId}/shared-memories/query) return past agreements. The knowledge graph (/api/knowledge/graphs/query) returns agent-specific context — what julia-agent owns, her past positions, what she agreed to last time.

This primes IntentDiscovery and OptionsGeneration with real context instead of stubs, leading to faster convergence and more relevant issue spaces.

Full Pipeline

End to end

From a single mycelium adapter add openclaw to agents reaching consensus — the whole flow is event-driven. No polling. Postgres NOTIFY pushes through SSE to the plugin which wakes the agent. NegMAS blocks on asyncio Futures bridged to room messages.

Every conversation feeds back into the knowledge fabric. CognitiveEngine gets smarter with every negotiation. The roadmap replaces the 5s poller with direct webhook push and adds coordination metrics collected per-round.
